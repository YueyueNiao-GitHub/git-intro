🔧

Git = 发动机（底层技术）

装在你电脑本地，记录每一次历史版本。写 Skill 时每个关键版本都想留一份，就是这个感觉。不需要联网，存在本地。

☁️

GitHub / GitLab = 车壳（云端平台）

在 Git 基础上搭建的云端协作平台。学会了 Git，换哪个平台都能用，操作逻辑完全一样。

快手内网 GitLab = 只有内部员工能访问，代码数据不出公司

04 核心概念，用大白话说
❯git glossary --plain
💾

存档

完成一个阶段就存一次，写清楚「这次做了什么」。就像给 Skill 的每次迭代打一个快照。

☁️

同步到云端

把本地存档上传到 GitHub，相当于「保存到网盘」。

⬇️

拉取最新

从 GitHub 下载别人的最新改动，开始工作前先做这一步，避免冲突。

🗂️

分支 = 草稿空间

复制一份 Skill 来做实验，主文件不动，做砸了直接扔掉，主文件毫发无损。

🔀

PR（提交审批）= 同事改完提交给你审核

同事在草稿空间做好了，提交 PR。你看完没问题点「批准合并」，他的内容才并入主文件。
核心流程：同事提交 → 你审批 → 合并

冲突怎么处理？
❯git merge
CONFLICT (content): Merge conflict in homepage.md
Automatic merge failed; fix conflicts and then commit.

你和同事 同时修改了同一个地方，Git 不知道保留谁的版本，就会提示冲突。

怎么解决

  • 保留你的版本
  • 保留同事的版本
  • 两边内容都保留，手动拼合

怎么避免

  • 每次工作前先拉取最新版本
  • 分工明确，各改各的部分
  • 及时合并，不让分支积压
